# whereEV2 지도 스타일 미니멀/모바일 친화 방향

배경 지도보다 **서비스 데이터(마커, 선택 상태, 하단 시트)**가 더 잘 보이도록 하는 방향 정리. **코드 수정 전** 적용 가능한 수준.

---

## 1. 문제 진단

### 1.1 현재 구조

| 항목 | 내용 |
|------|------|
| **베이스맵** | OSM 기본 타일 `https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png` |
| **렌더** | Leaflet + react-leaflet, 단일 TileLayer |
| **스타일 제어** | 타일 URL 교체만 가능. 타일 픽셀/레이어별 수정 불가 |

### 1.2 현재 지도가 주는 문제

| 문제 | 설명 |
|------|------|
| **배경이 과함** | OSM 기본 스타일은 공원(녹색), 수계(청색), 도로(다양한 굵기/색), POI·지명 등 정보가 많아 “알록달록”하고 시선을 뺏음. |
| **서비스 데이터가 묻힘** | 마커(파란 점)·선택 링·하단 시트가 지도 위에서 상대적으로 덜 돋보임. |
| **모바일에서 더 두드러짐** | 작은 화면에서 배경 지도 정보가 많을수록 “지도가 주인공”처럼 느껴짐. |
| **톤 불일치** | OSM 다채 색상과 whereEV2의 Gray + Single Blue 톤이 어울리지 않음. |

### 1.3 사용자 목표와의 관계

- 사용자는 **충전소를 빨리 찾고 판단**하는 것이 목표.
- 배경은 **위치/맥락만 제공**하면 되고, **마커·시트·선택 상태**가 주인공이어야 함.
- 따라서 “정돈·차분·한 톤·덜 자극적”이 whereEV2에 맞는 방향.

## 2. 현재 스택에서 가능한 것 / 어려운 것

### 2.1 Leaflet + Raster Tile 기준

| 구분 | 내용 |
|------|------|
| **가능** | **타일 URL 교체**로 다른 베이스맵 스타일로 전환. (Carto Positron, Stadia Alidade 등) |
| **가능** | **마커·팝업·시트**는 이미 우리가 제어 가능. 대비를 높이려면 배경을 “조용한” 타일로 바꾸면 됨. |
| **가능** | **CSS 필터**로 타일 전체에 `brightness`/`contrast`/`saturate` 등 적용해 다소 차분하게 만들 수 있음. (단, 모든 요소에 동일 적용, 선별 조정 불가) |
| **불가** | 타일 **레이어별**(도로/구획/배경) 색·두께·라벨 on/off 등 **개별 스타일 변경**. raster는 이미 렌더된 이미지라 불가. |
| **불가** | **줌 레벨별** 도로 두께·라인 정밀 조정. (특정 줌에서만 얇게 등) |
| **불가** | “한 가지 색상 톤으로 정리”를 **배경 타일 자체에서** 구현하려면, 그렇게 이미 렌더된 **다른 타일 공급자**를 선택하는 수밖에 없음. |

### 2.2 “색상 톤 한 가지 계열로”

- **가능**:  
  - **다른 베이스맵**을 쓰면 됨. (예: Carto Positron = 밝은 회색·흰색·연한 라인 한 톤)  
  - 또는 현재 OSM 타일 위에 **CSS `filter`**로 채도/명도만 낮춰 단조롭게 만들 수 있음. (모든 색에 동일 적용)
- **한계**:  
  - raster 한 장에 “도로만 파란 톤, 배경만 회색”처럼 **요소별 색 지정**은 불가.  
  - 정말 “한 톤”을 원하면 **이미 그렇게 만든 타일**을 쓰는 것이 현실적.

### 2.3 “구획/도로/배경 더 깔끔·얇게”

- **가능**:  
  - **미니멀 스타일을 제공하는 타일**로 교체하면 됨. (Positron, Stadia Alidade Smooth 등은 OSM보다 라인·라벨이 적고 단순함)  
  - **라벨 없음** 옵션(예: Carto `light_nolabels`)을 쓰면 글자 정보가 사라져 더 조용해짐.
- **한계**:  
  - “도로만 1px 더 얇게” 같은 **픽셀 단위 조정**은 raster 타일에서는 불가.  
  - “확대 전후 모두 선이 얇고 정돈되게”는 **해당 스타일을 제공하는 타일**을 고르는 것으로만 달성 가능.

### 2.4 “마커는 더 잘 보이게, 지도는 더 조용하게”

- **가능**:  
  - **배경을 미니멀 타일로 바꾸면** 대비로 인해 마커·시트가 상대적으로 잘 보임.  
  - 마커/팝업/시트는 우리가 계속 다듬을 수 있음. (테마 컬러, 테두리, 그림자 등)
- **한계**:  
  - 배경 “자체”를 우리가 세밀하게 조정하는 것은 raster 한계 내에서만 가능(타일 교체 + 필요 시 CSS filter).

## 3. Raster 타일 유지 시 가능한 베이스맵 교체안

모두 Leaflet `TileLayer`의 `url`·`attribution`만 바꾸면 됨. 기존 구조 유지.

| 안 | 타일 | 특징 | 비고 |
|----|------|------|------|
| **A** | **Carto Positron (light_all)** | 밝은 배경, 연한 회색 도로·경계, 라벨 있음. 단색에 가까움. | 무료(비상업 시). `light_nolabels`로 라벨 제거 가능. |
| **B** | **Carto Positron (light_nolabels)** | A와 동일하되 **글자 없음**. 지도가 가장 조용함. | 마커·시트가 가장 돋보이기에 적합. |
| **C** | **Carto Voyager (voyager_nolabels)** | Positron보다 약간 색이 있지만 OSM보다는 단순. 라벨 제거 가능. | “조금만 맵 느낌”을 남기고 싶을 때. |
| **D** | **Stadia Alidade Smooth** | 깔끔한 라인, 차분한 톤. | API 키 필요할 수 있음. |
| **E** | **CSS filter** | 현재 OSM 유지하고, 타일 컨테이너에 `filter: saturate(0.6) brightness(1.05)` 등 적용. | 구현은 가장 간단. 변화 폭은 제한적. |

**실제 URL 예 (Carto, 무료 사용 시)**  
- Positron with labels: `https://{s}.basemaps.cartocdn.com/light_all/{z}/{x}/{y}.png`  
- Positron no labels: `https://{s}.basemaps.cartocdn.com/light_nolabels/{z}/{x}/{y}.png`  
- subdomains: `'abcd'`  
- attribution: OSM + CARTO 명시 필요.

## 4. Vector tile / MapLibre 전환이 필요한 경우와 아닌 경우

| 목표 | Raster만으로 | Vector/MapLibre 필요 여부 |
|------|--------------|---------------------------|
| 배경을 “미니멀·한 톤”으로 | ✅ 가능 (Carto 등으로 교체) | ❌ 불필요 |
| 마커·시트가 더 잘 보이게 | ✅ 가능 (배경 조용히 + UI 강화) | ❌ 불필요 |
| 도로/구획 **요소별** 색·두께·라벨 on/off | ❌ 불가 | ✅ 필요 |
| 줌별로 선 두께·스타일 **세밀 제어** | ❌ 불가 | ✅ 필요 |
| 티맵/네이버와 **픽셀 단위로 동일** 복제 | ❌ 불가 | ✅ 필요(그래도 완전 동일은 어려움) |

**정리**  
- whereEV2 목표(“배경은 조용히, 서비스 데이터가 주인공”)는 **raster + 미니멀 타일 교체**로 충분히 달성 가능.  
- **Vector/MapLibre 전환**은 “요소별·줌별 세밀 스타일 제어”가 꼭 필요할 때만 검토.  
- 현재 규칙(기존 스택 유지, 불필요한 리팩토링 금지)을 고려하면 **우선 raster 범위에서 해결**하는 것이 맞음.

## 5. whereEV2에 맞는 지도 스타일 방향 3안

### 방향 1: Carto Positron no labels (가장 미니멀)

- **베이스맵**: Carto `light_nolabels`.
- **특징**: 밝은 회색·흰색, 연한 라인만. 글자 없음 → 지도가 가장 조용함. 마커·시트·선택 상태가 최대한 돋보임.
- **적합**: “지도 서비스보다 **충전소 찾기/판단**이 주인공”인 모바일 중심 서비스에 가장 잘 맞음.

### 방향 2: Carto Positron with labels (미니멀 + 지명)

- **베이스맵**: Carto `light_all`.
- **특징**: 1과 동일 톤이지만 도로명·지역명 등 라벨 있음. 위치/맥락은 더 읽기 쉽고, 여전히 OSM보다 훨씬 차분함.
- **적합**: “지명을 보면서 위치 확인”을 어느 정도 유지하고 싶을 때.

### 방향 3: 현재 OSM + CSS filter (최소 변경)

- **베이스맵**: OSM 유지.
- **추가**: 타일 레이어를 감싼 div에 `filter: saturate(0.7) brightness(1.05)` 등 적용해 채도만 낮춤.
- **특징**: 구현 부담 최소. 변화는 제한적이지만 “조금 덜 알록달록”하게는 가능.
- **적합**: 외부 타일 도입을 최대한 피하고, 당장은 소폭만 완화하고 싶을 때.

## 6. 추천안 1개와 이유

**추천: 방향 1 — Carto Positron no labels (`light_nolabels`)**

| 이유 | 설명 |
|------|------|
| **목표와 일치** | “배경보다 서비스 데이터가 주인공”을 가장 잘 만족. 라벨이 없어 지도가 가장 조용함. |
| **구현 부담 작음** | Leaflet 구조 그대로, TileLayer `url`·`attribution`·`subdomains`만 변경. |
| **모바일 친화** | 작은 화면에서도 배경이 덜 시끄럽고, 마커·하단 시트가 상대적으로 잘 보임. |
| **톤 정리** | 단색에 가까운 밝은 회색 톤으로, Gray + Single Blue 테마와 잘 어울림. |
| **규칙 준수** | Leaflet·raster 유지, 불필요한 리팩토링 없음. |

**추가 권장**  
- 적용 후, 마커·팝업·시트가 “충분히 돋보이는지”만 한 번 더 점검.  
- 필요하면 마커 테두리/그림자 강화, 시트 배경 대비 등 **UI 쪽 미세 조정**만 추가. (배경은 이미 조용해진 상태이므로)

## 7. 구현 우선순위

| 순서 | 작업 | 비고 |
|------|------|------|

## 9. 궁금 사항에 대한 답 (요약)

| 질문 | 답 |
|------|----|
| 지금 구조에서 **색상 톤을 한 가지 계열로** 정리할 수 있는지 | **가능.** 다만 “배경 타일 자체”를 우리가 레이어별로 다시 칠하는 것은 불가. **이미 한 톤으로 된 타일**(예: Carto Positron)로 **교체**하면 됨. |
| **구획/도로/배경**을 더 깔끔하게 만들 수 있는지 | **가능.** 미니멀 스타일 타일로 교체하면 됨. Raster에서는 “이 타일보다 1px 더 얇게” 같은 **픽셀 단위 조정**은 불가. |
| **확대 전후** 모두 선이 얇고 정돈되어 보이게 할 수 있는지 | **가능.** 그렇게 **이미 디자인된 타일**(Positron, Alidade 등)을 쓰면 됨. Raster는 줌별로 다른 스타일을 “우리가 지정”하는 것은 불가. |
| **마커를 더 잘 보이게** 하면서 **지도는 더 조용하게** 만들 수 있는지 | **가능.** ① 배경을 `light_nolabels` 등으로 교체해 지도를 조용하게 하고, ② 마커/팝업/시트는 기존처럼 우리가 스타일 제어(색, 테두리, 그림자)하면 됨. |

---

이 문서를 기준으로 **1단계는 TileLayer를 Carto Positron no labels로 교체**하고, 필요 시 마커·시트 가시성만 소폭 다듬으면, “미니멀·모바일 친화·서비스 데이터가 주인공” 방향을 raster 범위 안에서 달성할 수 있다.
