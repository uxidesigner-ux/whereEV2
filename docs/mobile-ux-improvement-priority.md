# whereEV2 모바일 UX 품질 개선 우선순위

현재 "동작하는 모바일 프로토타입"을 "실제로 쓰기 좋은 모바일 서비스"로 끌어올리기 위한 진단·우선순위. **코드 수정 전** 방향 정리용.

---

## 1. 현재 모바일 화면의 문제 진단

### 1.1 데스크탑 잔재처럼 느껴지는 부분

| 위치 | 현상 | 근거(코드/구조) |
|------|------|------------------|
| **시트 본문 상단** | "충전기 타입"이 라벨+chip 그리드로 먼저 나와, 폼/설정 화면처럼 보임 | App.jsx 517~535행: `<Typography variant="caption">충전기 타입</Typography>` + chip wrap. 데스크탑 패널과 동일한 라벨 체계. |
| **chip 크기·밀도** | 타입 chip이 작고(0.75rem, height 24), 여러 개 나열돼 데스크탑 필터 느낌 | 519~534행: `fontSize: '0.75rem', height: 24`. 모바일 터치 타겟(최소 44px) 미달. |
| **"가까운 순" chip** | 정보성만 있고 동작이 없어, 설정 토글처럼 보임 | 536~538행: 클릭 불가 Chip. "정렬 기준 표시"인데 필터처럼 보일 수 있음. |
| **"필터 더 보기"** | 작은 텍스트 버튼(0.75rem), 폼의 "고급 검색" 느낌 | 537행: `Button size="small"`, `fontSize: '0.75rem'`. 터치 영역 좁음. |
| **상세 모달** | 라벨+값 나열이 데스크탑 폼 스타일 | StationDetailModal.jsx 21~36행: `<strong>주소</strong> {address}` 형태. "판단용 요약"보다 "상세 폼" 인상. |
| **지도 Popup** | 텍스트가 많고 [상세 보기]만 강조, "한눈에 판단"보다 읽기 위주 | App.jsx 168~193행: statNm, 운영기관, 타입, 이용시간·전화, 주소, 버튼. 모바일에서는 요약 1~2줄+액션이 더 적합. |

### 1.2 모바일 첫 화면에서 정보 위계가 어색한 부분

| 순서 | 현재 위계 | 문제 |
|------|-----------|------|
| 1 | 시트 열면 "충전기 타입" 라벨 + chip들 | **필터가 1순위**로 보임. "지금 보이는 결과(목록)"가 먼저 와야 빠른 판단에 유리. |
| 2 | "가까운 순" + "필터 더 보기" | 정렬/필터가 목록보다 위. 목록을 먼저 보여주고, 필터는 접이거나 한 줄로 줄이는 편이 모바일스러움. |
| 3 | 충전소 목록 | 위에 필터 블록이 커서, 스크롤 없이 보이는 목록 개수가 적음. |
| 4 | 통계 보기(접기) | 적절히 후순위. |
| 5 | footer | 적절. |
| **닫힌 시트** | "현재 지도 영역 · N곳" 한 줄 | 좋음. 다만 시트를 열기 전 "할 수 있는 행동"이 지도 위에는 없음(현재 위치 버튼만). |

**정리**: 시트를 열었을 때 **"결과(목록) → 필터"** 순이 아니라 **"필터 → 결과"** 순이라, "몇 곳 중에서 고르기"보다 "설정하고 보기" 흐름에 가깝게 느껴짐.

### 1.3 리스트 카드 / 빠른 필터 / 상세 모달에서 판단 UX를 방해하는 부분

| 영역 | 문제 | 코드/위치 |
|------|------|-----------|
| **리스트 카드** | 카드가 작고(p: 1.25), 정보 밀도는 적절하나 터치 영역이 애매함 | StationListMobile.jsx: `p: 1.25`(10px). min-height 없음. 한 손으로 누르기엔 카드 높이가 다소 작을 수 있음. |
| | 선택 시 시각적 피드백만 있고, "지도에서 어디인지" 바로 연결되기 어려움 | 선택 시 flyTo + 상세 모달은 있으나, 리스트와 지도가 동시에 보이지 않아 "이 카드 = 지도 저 마커" 인지가 약함. |
| | 거리만 "약 1.2km"이고, "급속/완속" 등 타입이 텍스트라 한눈에 안 들어옴 | chgerTyLabel이 텍스트. 아이콘 또는 색으로 타입 구분 시 판단이 더 빠름. |
| **빠른 필터** | 타입 chip이 많을 때 가로 스크롤 없이 wrap되면 시트 상단이 길어짐 | 519~534행: flexWrap. 타입이 많으면 2~3줄 차지, 목록이 더 아래로 밀림. |
| | "필터 더 보기"가 모달이라, 적용 후 다시 시트로 돌아와야 해서 단계가 김 | 필터 더 보기 → 선택 → 적용 → 모달 닫기. 시트 안에 인라인으로 접기 블록이면 단계 감소. |
| **상세 모달** | "길찾기/전화"가 좋으나, 그 위 정보가 모두 동등한 비중으로 나열 | StationDetailModal: 주소·운영기관·타입·이용시간·전화가 동일 스타일. "지금 갈 수 있는지(이용시간)" "어디인지(주소)"가 더 강조되면 판단에 유리. |
| | 주소가 길면 한 줄에 다 나와 가독성 떨어짐 | address 한 줄. 줄바꿈/말줄임 없음. |
| **지도 Popup** | 내용이 많아서 "빠른 확인"보다 "읽기"에 가깝고, 상세 보기 전에 이미 많은 텍스트 | Popup 내 4줄+버튼. 모바일에서는 "이름 + 타입(또는 거리) + [상세 보기]" 수준이면 충분. |

### 1.4 통계/차트가 모바일에서 어느 정도까지 후순위여야 하는지

| 항목 | 현재 | 제안 |
|------|------|------|
| **위치** | 시트 본문에서 목록·footer 아래, "통계 보기" 아코디언 안 | 유지. 접기 기본이므로 후순위 노출은 적절함. |
| **아코디언 열었을 때** | StatCard 4개 + BarChart(높이 200) + PieChart(168) | 모바일에서는 **높이를 더 줄이거나** StatCard만 1줄 요약, 차트는 "간이 막대/도넛" 수준으로 축소해 스크롤 부담을 줄일 수 있음. |
| **역할** | "참고용 통계"이지 "행동 유도"가 아님 | 첫 화면·목록·상세에 통계를 끌어올리지 말 것. 현재처럼 "통계 보기" 안에만 두면 됨. |
| **조건부 노출** | 없음(항상 접기만) | 필터 결과 0건일 때 "통계 보기" 숨기기는 선택. 1차에서는 불필요할 수 있음. |

**정리**: 통계/차트는 **이미 후순위**로 잘 두었음. 개선 시에는 "아코디언 열었을 때 높이/정보량만 조금 줄이기" 정도면 충분.

### 1.5 지금 구조를 유지하면서 더 모바일스럽게 만들 수 있는 개선점

| 개선 | 설명 | 구조 유지 여부 |
|------|------|----------------|
| **시트 본문 순서** | "목록 먼저" 또는 "요약 1줄 + 목록" → 그 다음 "필터" 접기/한 줄 | 시트·지도·상세 모달 구조 유지. 블록 순서만 변경. |
| **필터 영역 압축** | 타입을 "전체/급속/완속" 등 2~3개로 묶거나, 가로 스크롤 한 줄로 고정해 상단 높이 확보 | chip 개수·라벨만 조정. filterChgerTy 로직 유지. |
| **리스트 카드** | min-height 48px 이상, padding 확대. 타입을 아이콘/뱃지로 표시 | StationListMobile 스타일만. props/데이터 구조 동일. |
| **상세 모달** | 주소·이용시간을 상단 강조, 나머지 보조. 주소 말줄임/줄바꿈 | StationDetailModal 레이아웃만. |
| **Popup 단순화** | 모바일일 때 Popup 내용을 "이름 + 타입(또는 거리) + [상세 보기]"로 제한 | MapView에서 isMobile 분기 또는 Popup 내용만 조건부. |
| **시트 peek** | 닫힌 상태에서도 리스트 1~2개 살짝 보이게(peek) 하면 "열면 목록이 있다"가 더 명확 | 시트 높이를 완전 56px이 아니라 56+α로 하거나, 헤더 아래 1줄만 노출. 선택 사항. |
| **터치 타겟** | chip·버튼·리스트 카드 최소 44px 높이 | sx 수치만 조정. |
| **테스트 배너** | "whereEV2 TEST BUILD" 고정 배너는 개발용. 배포 시 제거 또는 조건부 | 제거/환경변수 분기만 하면 됨. |

## 3. 반드시 바로 손봐야 할 것

- **시트 본문 순서**: 시트를 열었을 때 **목록(또는 요약 1줄+목록)을 필터보다 위**에 두기. 정보 위계를 "결과 → 필터"로 맞추는 것이 가장 영향이 큼.
- **리스트 카드 터치 영역**: 카드에 `minHeight: 48` 이상, `padding` 확대해 한 손으로 누르기 편하게.
- **chip·필터 더 보기 버튼 터치 타겟**: `height: 24` → 44px 이상(또는 padding으로 터치 영역만 44px 이상).
- **상세 모달**: 주소·이용시간을 상단에 크게, 나머지는 보조. 주소는 `whiteSpace`, `overflow`, `textOverflow`로 줄바꿈/말줄임 처리.

## 4. 나중으로 미뤄도 되는 것

- 시트 peek(닫힌 상태에서 리스트 1줄 노출): UX 개선이지만 구조 변경이 있어 2차로.
- 리스트 카드에 타입 아이콘/뱃지: 판단 속도에 도움되나, 현재 텍스트만으로도 동작함. 2차.
- "필터 더 보기"를 시트 내 인라인 접기로 변경: 단계 감소에 유리하나, 모달 유지해도 됨. 2차.
- 통계 아코디언 내 차트 높이 축소: 이미 접기로 후순위라 급하지 않음.
- 테스트 배너: 배포 전에만 정리하면 됨.
- 지도 Popup 단순화: 개선이지만, 상세 보기로 이어지므로 우선순위는 3·4번 다음.

---

## 5. 2차 UI/UX 개선 범위 제안

- **필터 UX**: 타입을 "급속/완속/전체" 등으로 묶거나, 시트 내 "필터" 접기 블록으로 모아 상단 고정 영역 줄이기.
- **리스트·지도 연동 인지**: 목록 선택 시 지도가 화면에 같이 보이도록 시트 높이를 조절하거나, 카드에 "지도에서 보기" 힌트 추가(선택).
- **Popup 단순화**: 모바일일 때 Popup을 이름+타입(또는 거리)+[상세 보기]만 노출.
- **통계 섹션**: 아코디언 열었을 때 StatCard 1줄 요약 + 차트 높이 축소.
- **접근성**: aria-label, 포커스 순서 점검(1차에서 터치 타겟만 해도 큰 개선).
